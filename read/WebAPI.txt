Web API The Good Parts

🔵 1章 Web APIとは何か

まとめ

🔴 Web APIを美しく設計する重要性
・設計の美しいWebAIは使いやすい
・設計の美しいWebAPIは変更しやすい
・設計の美しいWebAPIは頑強である
・設計の美しいWebAPIは恥ずかしくない

🔴 WebAPIを美しくするには
・仕様が決まっているものに関しては仕様に従う
・仕様が存在してないもの関しては、デファクトスタンダードに従う

🔵 2章 APIエンドポイントの考え方

まとめ

🔴エンドポイントの基本的な設計
・短く入力しやすいURI
・人間が読んで理解しやすURI
・大文字小文字が混在していないURI
・改造しやすいURI
・サーバー側のアーキテクチャーが反映されないURI
・ルールが統一されたURI

🔴クエリパラメータとパスの使い分け
・一意なリソースを表すのに必要な情報かどうか
・省略可能かどうか
